# NewLife.Remoting 需求文档

> 版本：v1.0 | 日期：2026-07-15
>
> 完成状态在[功能清单](功能清单.md)追踪，详细设计在[架构设计](架构设计.md)展开。

## 1. 背景与愿景

### 1.1 系统定位

NewLife.Remoting 是 NewLife 技术栈的统一远程通信基础设施，由两个 NuGet 包组成：

| 包 | 定位 | 覆盖 |
|:---|:-----|:------|
| **NewLife.Remoting** | 核心通信库 | RPC 客户端/服务端、协议编解码、连接管理、流式调用、指令下发管线、客户端基类、自动更新 |
| **NewLife.Remoting.Extensions** | 扩展库 | 设备管理控制器、WebSocket/SSE 会话、Token 鉴权、命令响应总线 |

### 1.2 愿景

使 .NET 开发者能在 net45 ~ net10.0 单一生态内完成全部远程通信需求——RPC（二进制高性能）、HTTP/REST、WebSocket 长连接、SRMP 远程消息协议，同时为 IoT 场景提供指令下发、设备管理和自动更新的一站式解决方案。

## 2. 核心目标

| 编号 | 目标 | 层级 | 描述 |
|:----|:-----|:----:|------|
| M1 | RPC 核心框架 | 核心层 | 跨平台、多协议、高性能远程过程调用基础设施 |
| M2 | 流式调用 | 核心层 | Server-Streaming：Controller 返回 `IAsyncEnumerable<T>` 自动逐条推送，SRMP + SSE 双协议 |
| M3 | 元数据传递 | 核心层 | `ApiClient.Headers` 字典透传 TraceId/TenantId，服务端通过 ControllerContext 读取 |
| M4 | 指令下发 | 扩展层 | 面向 IoT 的指令下发管线，事件总线广播 + WS/SSE/Ping 多通道 |
| M5 | 客户端基类 | 核心层 | 应用客户端通用基类，封装登录/心跳/升级/指令/事件完整生命周期 |
| M6 | 自动更新 | 核心层 | 安全跨平台应用自动更新：检测→下载→校验→解压→覆盖→重启 |
| M7 | 重试策略 | 核心层 | 可插拔瞬时错误重试：`IRetryPolicy` 接口 + `MaxRetries` + 连接切换 |

## 3. 功能需求

### 3.1 M1 RPC 核心框架

- **M1-1 ApiClient 客户端**：TCP/UDP/HTTP 多协议客户端，自动连接管理、故障转移、401 自动重登
- **M1-2 ApiServer 服务端**：多协议监听，请求处理流水线，支持 Controller 和委托模式
- **M1-3 ApiAction 动作元数据**：反射预编译加速，支持二进制参数 Packet、流式返回 IAsyncEnumerable、Accessor 参数检测
- **M1-4 ApiHandler 处理器**：请求路由到 Controller，ControllerContext 构建与生命周期管理
- **M1-5 ApiManager 动作管理**：控制器注册与发现，类型注册和实例注册
- **M1-6 Encoder 编解码**：JSON 默认编码，支持替换 IEncoder，二进制 Packet 直通
- **M1-7 Cluster 连接管理**：单连接/连接池两种模式，故障转移、平滑切换
- **M1-8 消息模型**：SRMP 协议消息模型 Flag/Seq/Length/Payload
- **M1-9 内存管理**：IOwnerPacket 零拷贝链式传递，ArrayPool 池化，级联释放
- **M1-10 会话级别 Encoder**：每个会话可独立设置 Encoder

### 3.2 M2 流式调用 (Server-Streaming)

- **M2-1 SRMP 协议 Streaming Flag**：Flag 字节 bit5(Streaming) + bit4(EndOfStream) 标记流式帧
- **M2-2 ApiAction.IsStreaming 检测**：反射检测返回类型自动标记流式
- **M2-3 Controller 流式检测**：Process 中通过接口检测 IAsyncEnumerable 自动启用流式
- **M2-4 InvokeStreamAsync 客户端**：返回 `IAsyncEnumerable<T>`，支持 CancellationToken 中途取消
- **M2-5 ClientBase 代理**：ClientBase 代理 InvokeStreamAsync
- **M2-6 ProcessStream 服务端**：泛型 Helper 处理值类型/引用类型流，异常时发送错误帧
- **M2-7 Encoder 流式块编码**：`EncodeStreamChunk` 预计算长度，SpanWriter 合并写入
- **M2-8 HTTP SSE 兼容**：输出 `text/event-stream`，浏览器和 curl 可调试
- **M2-9 结束标记帧**：空 action + 空 data 标记流结束
- **M2-10 客户端断开自动停止**：服务端通过 CancellationToken 感知断开，自动停止枚举

### 3.3 M3 元数据/Headers 传递

- **M3-1 ApiClient.Headers 属性**：`IDictionary<String, String?>` 属性
- **M3-2 InvokeAsync 自动注入**：调用时将 Headers 合并到参数字典
- **M3-3 流式调用自动注入**：InvokeStreamAsync 同样自动注入 Headers
- **M3-4 TraceId 自动注入**：DefaultSpan.Current?.TraceId 非空时自动注入
- **M3-5 服务端提取**：ApiHandler.Prepare 将参数字典放入 `ctx.Items["Headers"]`
- **M3-6 ControllerContext 读取**：`ControllerContext.Current.Items["Headers"]` 可读取

### 3.4 M4 指令下发

- **M4-1 SessionManager 会话管理器**：会话 CRUD、命令下发、响应等待、响应广播完整闭环
- **M4-2 事件总线命令发布**：`PublishAsync` 通过 EventBus 广播 `code#json`
- **M4-3 响应等待与回调注册**：TCS 回调注册，超时自动清理，支持 CancellationToken
- **M4-4 响应广播管线**：设备 CommandReply 通过 EventBus ResponseTopic 广播
- **M4-5 CommandSession 基类**：通用会话基类，管理 Code/Active/SetOnline
- **M4-6 WsCommandSession**：WebSocket 长连接命令下发
- **M4-7 SseCommandSession**：SSE 轻量降级通道，WS 不可用时的替代方案
- **M4-8 IDeviceService 接口**：设备 CRUD、登录/注销/心跳/命令/事件/升级
- **M4-9 Token 鉴权**：JWT 令牌验证，区分应用令牌和设备令牌
- **M4-10 BaseDeviceController**：HTTP 端点 SendCommand/CommandReply/Notify/NotifySSE/Upgrade
- **M4-11 向后兼容降级**：未注入 ICommandResponseBus 时自动降级到旧 cmdreply 队列

### 3.5 M5 客户端基类

- **M5-1 核心生命周期**：打开/关闭/重连，Features 标志位控制功能开关
- **M5-2 登录/注销**：自动登录、自动重登、Token 管理
- **M5-3 心跳 Ping**：定时心跳，搭载积压命令下发，更新在线状态
- **M5-4 命令接收与执行**：`ReceiveCommand` → `OnReceiveCommand` → `ExecuteCommand` 委托分发
- **M5-5 Commands 委托注册**：Commands 字典注册命令名到处理委托
- **M5-6 WebSocket 通道**：WS 连接管理、自动重连、心跳保活
- **M5-7 SSE 通道**：SSE 降级通道，WS 不可用时自动启用
- **M5-8 事件上报**：批量上报设备事件
- **M5-9 自动升级**：全链路更新：检测→下载→校验→解压→覆盖→重启
- **M5-10 流式调用代理**：代理底层 RPC 客户端的 InvokeStreamAsync

### 3.6 M6 自动更新

- **M6-1 文件操作**：下载/解压/文件覆盖/回滚，先改名再拷贝策略
- **M6-2 版本去重**：`_lastVersion` 避免重复更新同一版本
- **M6-3 下载与哈希校验**：HTTP 下载 .zip，MD5/SHA256 校验
- **M6-4 解压与覆盖**：安全替换运行中文件，Linux sync 强制落盘
- **M6-5 跨平台重启**：Windows/Linux/OSX 拉起新进程，支持 systemd
- **M6-6 回滚保护**：更新失败时自动恢复被移动的旧文件

### 3.7 M7 重试策略

- **M7-1 IRetryPolicy 接口**：`ShouldRetry` 按异常类型和重试次数决策
- **M7-2 MaxRetries 配置**：不含首次尝试，0 表示禁用
- **M7-3 指数退避延迟**：策略实现自行控制退避算法
- **M7-4 连接切换**：refreshClient 为 true 时更换底层连接再重试

## 4. 🧊 暂缓清单

| 功能 | 暂缓理由 | 解冻前提 |
|:-----|:---------|:----------|
| 客户端流式 (Client Streaming) | Server-Streaming 已覆盖 IoT 推送场景 | 出现明确的客户端→服务端流式需求 |
| 双向流式 (Bidirectional Streaming) | WebSocket 长连接已提供类似能力 | WebSocket 不满足的场景出现 |
| Protobuf 支持 | 保持 JSON + Packet 二进制直通，避免引入新序列化 | 跨语言互操作成为强需求 |
| UDP 流式调用 | UDP 无连接，流式语义不适合 | 无 |
| 组件图/时序图自动生成 | Mermaid 文本图已满足文档需求 | 文档规模扩大需要自动生成 |

## 5. 不做什么

- 不改 `IMessage` / `IActionFilter` / `ICluster` 等 NewLife.Core 基础接口
- 不新增外部 NuGet 依赖
- 不替换现有 JSON 序列化方案
- 不引入 gRPC 或 SignalR 作为依赖
