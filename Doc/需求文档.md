# 需求文档 — NewLife.Remoting

> 最后更新: 2026-07-15

## 愿景

**统一高性能远程通信基础设施**：提供一套覆盖 RPC（二进制高性能）、HTTP/REST、WebSocket 长连接、SRMP 远程消息协议的集成通信框架，同时为 IoT 场景提供指令下发、设备管理和自动更新能力，使 .NET 开发者能在 net45 ~ net10.0 单一生态内完成全部远程通信需求。

---

## 系统定位

NewLife.Remoting 是 NewLife 技术栈的统一远程通信基础设施，由两个 NuGet 包组成：

| 包名 | 定位 | 核心覆盖 |
|------|------|---------|
| **NewLife.Remoting** | 核心通信库 | RPC 客户端/服务端、协议编解码、连接管理、流式调用、指令下发管线、客户端基类、自动更新 |
| **NewLife.Remoting.Extensions** | 扩展库 | 设备管理控制器、WebSocket/SSE 会话、Token 鉴权、命令响应总线 |

---

## 用户角色

| 角色 | 说明 | 核心诉求 |
|------|------|---------|
| 后端开发者 | 使用 Remoting 构建微服务 | 高性能 RPC 调用、流式推送数据、链路追踪 ID 透传 |
| IoT 设备开发者 | 通过 Remoting 连接设备 | 稳定长连接、指令下发与执行、自动更新 |
| 系统架构师 | 评估 RPC 框架选型 | 对标 gRPC，了解差异和优势，评估适用场景 |

## 功能需求

### M1: RPC 核心框架

> 提供跨平台、多协议、高性能的远程过程调用能力

| 编号 | 功能 | 优先级 | 描述 |
|------|------|:------:|------|
| M1-1 | ApiClient 客户端 | 🔴 Must | 支持 TCP/UDP/HTTP 多协议客户端，自动连接管理、故障转移、401 自动重登 |
| M1-2 | ApiServer 服务端 | 🔴 Must | 多协议监听，请求处理流水线，支持 Controller 模式和委托模式 |
| M1-3 | ApiAction 动作元数据 | 🔴 Must | 反射预编译加速，支持二进制参数(Packet)、流式返回(IAsyncEnumerable)、Accessor 参数检测 |
| M1-4 | ApiHandler 处理器 | 🔴 Must | 请求路由到 Controller，ControllerContext 构建与生命周期管理 |
| M1-5 | ApiManager 动作管理 | 🔴 Must | 控制器注册与发现，支持类型注册和实例注册 |
| M1-6 | Encoder 编解码 | 🔴 Must | JSON 默认编码，支持替换为自定义 IEncoder，二进制 Packet 直通 |
| M1-7 | Cluster 连接管理 | 🔴 Must | 单连接/连接池两种模式，故障转移、平滑切换 |
| M1-8 | 消息模型 | 🔴 Must | SRMP 协议消息模型 Flag/Seq/Length/Payload |
| M1-9 | 内存管理 | 🔴 Must | IOwnerPacket 零拷贝链式传递，ArrayPool 池化，级联释放 |
| M1-10 | 会话级别 Encoder | 🟡 Should | 每个会话可独立设置 Encoder |

### M2: 流式调用 (Server-Streaming)

> Controller 返回 `IAsyncEnumerable<T>` 时自动流式推送，SRMP + HTTP SSE 双协议兼容

| 编号 | 功能 | 优先级 | 描述 |
|------|------|:------:|------|
| M2-1 | SRMP 协议 Streaming Flag | 🔴 Must | Flag 字节 bit5(Streaming) + bit4(EndOfStream) 标记流式帧 |
| M2-2 | ApiAction.IsStreaming 检测 | 🔴 Must | 反射检测返回类型自动标记流式 |
| M2-3 | Controller 流式检测 | 🔴 Must | Process 中通过接口检测 IAsyncEnumerable 自动启用流式 |
| M2-4 | InvokeStreamAsync 客户端 | 🔴 Must | 返回 IAsyncEnumerable\<T\>，支持 CancellationToken 中途取消 |
| M2-5 | ClientBase 代理 | 🔴 Must | ClientBase 新增 InvokeStreamAsync 代理方法 |
| M2-6 | ProcessStream 服务端 | 🔴 Must | 泛型 Helper 处理值类型/引用类型流，异常时发送错误帧 |
| M2-7 | Encoder 流式块编码 | 🔴 Must | EncodeStreamChunk 预计算长度，SpanWriter 合并写入 |
| M2-8 | HTTP SSE 兼容 | 🟡 Should | 输出 text/event-stream，浏览器和 curl 可调试 |
| M2-9 | 结束标记帧 | 🔴 Must | 空 action + 空 data 标记流结束 |
| M2-10 | 客户端断开自动停止 | 🔴 Must | 服务端通过 CancellationToken 感知客户端断开，自动停止枚举 |

### M3: 元数据/Headers 传递

> ApiClient.Headers 字典自动传递 TraceId/TenantId，服务端通过 ControllerContext 读取

| 编号 | 功能 | 优先级 | 描述 |
|------|------|:------:|------|
| M3-1 | ApiClient.Headers 属性 | 🔴 Must | IDictionary\<String, String?\> 属性 |
| M3-2 | InvokeAsync 自动注入 | 🔴 Must | 调用时将 Headers 合并到参数字典 |
| M3-3 | 流式调用自动注入 | 🔴 Must | InvokeStreamAsync 同样自动注入 Headers |
| M3-4 | TraceId 自动注入 | 🔴 Must | 若 DefaultSpan.Current?.TraceId 非空，自动加入 Headers["TraceId"] |
| M3-5 | 服务端提取 | 🔴 Must | ApiHandler.Prepare 将参数字典放入 ctx.Items["Headers"] |
| M3-6 | ControllerContext 读取 | 🔴 Must | ControllerContext.Current.Items["Headers"] 可读取 |

### M4: 指令下发

> 面向 IoT 和设备的指令下发管线，支持 WebSocket 主通道 + SSE 降级，集群精确路由

| 编号 | 功能 | 优先级 | 描述 |
|------|------|:------:|------|
| M4-1 | SessionManager 会话管理器 | 🔴 Must | 会话 CRUD、命令下发、响应等待、响应广播完整闭环 |
| M4-2 | 事件总线命令发布 | 🔴 Must | 通过 EventBus 广播 code#json 消息 |
| M4-3 | 响应等待与回调注册 | 🔴 Must | TCS 回调注册，超时自动清理，支持 CancellationToken |
| M4-4 | 响应广播管线 | 🔴 Must | 设备 CommandReply 通过 EventBus ResponseTopic 广播 |
| M4-5 | CommandSession 基类 | 🔴 Must | 通用会话基类，管理 Code/Active/SetOnline |
| M4-6 | WsCommandSession | 🔴 Must | WebSocket 长连接命令下发 |
| M4-7 | SseCommandSession | 🟡 Should | SSE 轻量降级通道，WS 不可用时的替代方案 |
| M4-8 | 设备路由表 | 🟡 Should | Redis Hash 全局路由表，跨节点精确转发 |
| M4-9 | IDeviceService 接口 | 🔴 Must | 设备 CRUD、登录/注销/心跳/命令/事件/升级 |
| M4-10 | Token 鉴权 | 🔴 Must | JWT 令牌验证，区分应用令牌和设备令牌 |
| M4-11 | BaseDeviceController | 🔴 Must | HTTP 端点：SendCommand/CommandReply/Notify/NotifySSE/Upgrade |
| M4-12 | 向后兼容降级 | 🔴 Must | 未注入 ICommandResponseBus 时自动降级到旧 cmdreply 队列 |

### M5: 客户端基类

> 应用客户端通用基类，封装登录/心跳/升级/指令/事件等完整设备通信流程

| 编号 | 功能 | 优先级 | 描述 |
|------|------|:------:|------|
| M5-1 | 核心生命周期 | 🔴 Must | 打开/关闭/重连，Features 标志位控制功能开关 |
| M5-2 | 登录/注销 | 🔴 Must | 自动登录、自动重登、Token 管理 |
| M5-3 | 心跳 Ping | 🔴 Must | 定时心跳，搭载积压命令下发，更新在线状态 |
| M5-4 | 命令接收与执行 | 🔴 Must | ReceiveCommand → OnReceiveCommand → ExecuteCommand 委托分发 |
| M5-5 | Commands 委托注册 | 🔴 Must | Commands 字典注册命令名到处理委托的映射 |
| M5-6 | WebSocket 通道 | 🔴 Must | WS 连接管理、自动重连、心跳保活 |
| M5-7 | SSE 通道 | 🟡 Should | SSE 降级通道，WS 不可用时自动启用 |
| M5-8 | 事件上报 | 🔴 Must | 批量上报设备事件 |
| M5-9 | 自动升级 | 🔴 Must | 全链路自动更新：检测→下载→校验→解压→覆盖→重启 |
| M5-10 | 流式调用代理 | 🔴 Must | 代理底层 RPC 客户端的 InvokeStreamAsync |

### M6: 自动更新

> 安全可靠的跨平台应用自动更新机制

| 编号 | 功能 | 优先级 | 描述 |
|------|------|:------:|------|
| M6-1 | 文件操作 | 🔴 Must | 下载/解压/文件覆盖/回滚，先改名再拷贝策略 |
| M6-2 | 版本去重 | 🔴 Must | _lastVersion 避免重复更新同一版本 |
| M6-3 | 下载与哈希校验 | 🔴 Must | HTTP 下载 .zip，MD5/SHA256 校验 |
| M6-4 | 解压与覆盖 | 🔴 Must | 安全替换运行中文件，Linux sync 强制落盘 |
| M6-5 | 跨平台重启 | 🔴 Must | Windows/Linux/OSX 拉起新进程，支持 systemd 服务 |
| M6-6 | 回滚保护 | 🟡 Should | 更新失败时自动恢复被移动的旧文件 |

### M7: 重试策略

> 可插拔的瞬时错误重试机制

| 编号 | 功能 | 优先级 | 描述 |
|------|------|:------:|------|
| M7-1 | IRetryPolicy 接口 | 🟡 Should | ShouldRetry 按异常类型和重试次数决策 |
| M7-2 | MaxRetries 配置 | 🟡 Should | 不含首次尝试，0 表示禁用 |

## 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | RPC 单次延迟 ~41μs（端到端含 TCP 往返 + JSON）；典型吞吐 10 万 TPS；流式首帧延迟与普通 RPC 一致 |
| **兼容性** | net45 ~ net10.0 多框架目标；旧客户端可连接新服务端；新客户端可连接旧服务端（非流式请求） |
| **安全** | JWT 令牌验证；数据库异常脱敏；401 自动重登 |
| **可观测性** | ILog 日志；ITracer 链路追踪；ICounter 统计；慢调用阈值告警 |
| **稳定性** | ArrayPool 零拷贝内存管理；级联释放防止泄漏；重试策略处理瞬时故障 |

## 暂缓清单

| 功能 | 原因 |
|------|------|
| 客户端流式 (Client Streaming) | 当前无明确需求，Server-Streaming 即可覆盖 IoT 推送场景 |
| 双向流式 (Bidirectional Streaming) | 复杂度高，WebSocket 长连接已提供类似能力 |
| Protobuf 支持 | 不引入新序列化方案，保持 JSON + Packet 二进制直通 |
| UDP 流式调用 | UDP 天然无连接，流式语义不适合 |
| 组件图/时序图自动生成 | 需专用绘图工具，文档中已含 Mermaid 文本图 |

---

## 不做什么

- 不改 `IMessage` / `IActionFilter` / `ICluster` 等 NewLife.Core 基础接口
- 不新增外部 NuGet 依赖
- 不替换现有 JSON 序列化方案
- 不引入 gRPC 或 SignalR 作为依赖

## 术语表

| 术语 | 定义 |
|------|------|
| SRMP | Simple Remote Message Protocol，NewLife 自研 RPC 协议 |
| SSE | Server-Sent Events，HTTP 单向流式推送标准 |
| Server-Streaming | 服务端流式调用：一次请求，多次响应 |
| Headers | 元数据键值对，在 RPC 调用中透传 |
| IOwnerPacket | 拥有缓冲区管理权的数据包，基于 ArrayPool，支持链式级联释放 |
| EventBus | 事件总线，支持内存（进程内）和 Redis（跨进程）模式 |
| SessionManager | 会话管理器，管理 WebSocket 长连接会话生命周期 |
